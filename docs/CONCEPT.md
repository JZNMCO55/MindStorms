# MindStorms — 概念与设计文档

> 状态:概念阶段(2026-06)。本文是讨论沉淀,会持续迭代。代码尚未开始。

## 1. 一句话定位

把一个人的思维方式「蒸馏」成多张有性格的**角色卡**——不同的你、你生命里的人、乃至名人——让他们在同一个聊天窗口里依次发言、互相反驳,由「主宇宙的我」收敛成决策建议或一篇可发表的记录。

灵感来自动漫里「头脑内部多个自我讨论」的经典场景,把它具象成真实可用的工具。

## 2. 核心原语:角色卡(Card)

所有议会成员——无论来源——都是同一种对象:**角色卡**。「召集/点将/花名册/关系表/养成」都是对角色卡的操作。这是整个系统得以实现的拱顶石。

数据结构草图:

```
Card {
  id
  name              // "没放弃画画的我" / "我印象里的老王" / "稻盛和夫"
  source            // self_facet | self_parallel | self_mirror | relation | figure
                    // self_mirror = 全部语料整合出的最高保真的你;只在镜子模式用;职责是"只问不答"
  fidelity          // high | mid | low —— 一等字段,驱动行为(见 §4、§8)
  divergence_point? // 仅平行自我:在哪个人生岔路分出去
  competence[]      // 习得的技能 / 擅长的视角领域
  utility           // 优化什么、愿意牺牲什么(效用函数)
  time_horizon      // 时间视野
  voice             // 说话风格、口头禅
  grounding[]       // 锚点素材:引用的真实原话/事例(高保真卡才有;本地存储)
  provenance_label  // 显示用:"蒸馏自你" / "你印象里的他" / "公众形象漫画化"
  publish_policy    // 由 fidelity 派生:freely | abstract_only | gated
  history           // 养成/纠错记录
}
```

## 3. 角色卡的四个来源(三条正交轴 + 名人)

- **横切 — 当下你的内部切面**:理性 / 感性 / 野心 / 保守。处理「我现在**怎么感受**这件事」的价值冲突。
- **纵切 — 平行宇宙的你**:在人生岔路上分叉出去的版本(去创业的你、留学术界的你、没放弃画画的你)。每个 = `分叉点 × 习得的技能 × 由此长出的主导气质`,把抽象效用函数具象成「另一种人生」。处理重大、不可逆决策。
- **关系切 — 你内心建模的他人**:从聊天记录里天然浮现的关系图。注意:蒸馏的是**「你心里的他」**,不是真人(详见 §7)。
- **名人/历史人物**:整批引进一个不属于你的世界观来「启发主宇宙的我」,是对回音室问题最干净的解药。

## 4. 保真度光谱(系统自洽的关键)

| 来源 | 真实是什么 | 保真度 | 最适合 | 发表风险 |
|---|---|---|---|---|
| 蒸馏的「我」 | 你自己 | 高 | 决策 + 探索 | 低 |
| 我印象里的朋友 | 你的感知 | 中 | 关系决策、社会后果模拟 | 高(真人具名) |
| 历史人物 | 文化原型 | 低 | 探索、启发 | 低 |
| 在世名人 | 公众形象漫画化 | 最低 | 启发、撞观点 | 高(在世具名) |

保真度越高 → 越靠决策模式、现实性旋钮拧紧;越低 → 越靠探索模式、旋钮松开。**名人卡不可当事实权威**。

## 5. 花名册与点将(casting)

折中方案:维护一个随蒸馏不断长大的**全体角色卡花名册**;但每场会议从中**点将组阁**(上场阵容)。既有连续养成,又每场精准。「召集」这个动作本身是产品的仪式感来源——相当于「组一手牌」。

**世界生成与收藏(2026-06-13 UI 决策,取代"性格切面"做默认阵容):**
- 默认阵容用**平行世界的我**——每个带**世界编号**(如 #C-17)、**经历**、以及由经历长出的**性格**;不再用通用的理性/野心/感性切面(那是满大街"六顶思考帽"的版本)。横切的性格没消失,而是变成**每个世界由经历长出的属性**。
- 机制:**生成新世界**(按话题/随机)→ 看 ta 的经历与性格 → 满意就**加入收藏**(角色卡库),可增删创建。这就是 gacha/召唤 + 养成的留存玩法。
- **主世界的我**是所有世界的分叉**基点**与生成**种子**(= 蒸馏出的真实的你)。随机世界是探索/灵感级的**"虚构可能"**,做决策时主世界的我要对它们打折(见 §9 与 [[distillation-pitfalls]] 坑2)。
- UI 落地:左栏底部「角色卡库」标签管理收藏;右栏点角色看世界详情;每场对话各自记住类型(决策/探索/镜子)与阵容。

## 6. 三种模式(同一引擎三套配置,三模式并重)

区分模式的真正的轴是**灯光打向谁**:前两种议会**向你输出**,第三种议会**从你抽取**。

| | 决策模式 | 探索模式 | 镜子 / 对话模式 |
|---|---|---|---|
| 形态 | **N 张卡互相辩论** | **N 张卡互相辩论** | **你 ↔ 单个镜子自我的 1:1 对话** |
| 谁在主导 | 替你**收敛** | 替你**发散** | **引导你**(只问不答,苏格拉底接生婆) |
| 灯光 | 向你输出 | 向你输出 | **从你抽取** |
| 收敛目标 | 收敛到被回避的取舍 + 建议 | 刻意不收敛,把多种看法并置 | 不给答案,把你自己的答案引导出来 |
| 你的角色 | 召集者 / 观众 | 召集者 / 观众 | **抛出问题、亲自作答** |
| 主力卡 | 平行自我(拧紧现实性) | 越野越好(松开现实性) | **镜子自我 `self_mirror`**(最高保真的整合你) |
| 典型场景 | 职业/人生重大抉择 | 哲学、对某事的看法、创意脑暴 | 想清楚一件事,尤其**被忽略的人际关系** |

镜子模式的前提是「只有自己最了解自己」——答案在你心里,镜子自我只负责把它引导出口。它还顺带是**检验蒸馏保真度的最好场子**:对话像不像你,你一聊就知道。

## 7. 议会运行与「主宇宙的我」

- **主宇宙的我 = 主持人 / 最终决策者 / 书记员**:唯一承担后果的人,负责整合众声、点出被回避的取舍、产出文章;也是纠错反馈的来源。
- 流程:抛出问题 → 上场角色卡依次发言、互相反驳(可多轮)→ 主宇宙收敛。
- **决策模式的交付物不是辩论本身,是「被你回避的那个取舍」**:不是「建议你做 X」,而是「真正的选择是——哪一种损失你能承受」。

## 8. 公众号飞轮(自我强化闭环)

```
召集"我"们讨论 → 主宇宙口吻综合成第一人称文章 → 发公众号
        ↑                                            │
        └─── 这篇文章(经你编辑认领后)回灌蒸馏 ←──────┘
```

公众号一身三任:**留存 + 获客 + 蒸馏的新语料**。「以主宇宙口吻改写」这步同时是**隐私/策展滤网**。

## 9. 隐私与伦理护栏

1. **关系切/名人卡显式标注**(「我印象里的 XX」「公众形象」),视觉上与你自己的「我」区分。
2. **发表 gate**:关系切与在世名人的内容若要发表,必须**先抽象化、去标识、明确虚构**(「我身边一个很稳的朋友会提醒我……」),绝不带真名原话出街(名誉/隐私真风险)。
3. **本地优先**:原始聊天记录(含好友数据,受 PIPL 约束)**死锁本地、加密、永不上云**;上云的只有高度抽象的画像 + 当前问题。
4. **防人格「模型坍缩」**:用户亲手写的原文永远高权重当 ground truth;AI 综合的文章只算草稿,**必须经用户编辑认领后**才有资格回灌蒸馏。
5. **保真度驱动行为**:fidelity 决定决策权重、发表策略、默认模式——安全与认识论都挂在卡上。
6. 私有卡(你/朋友)永不出本机、不可分享;公共预设卡(名人/原型)可做成共享卡库(增长/变现口子)。

## 10. 蒸馏 pass(一次性、可重跑)

- **输入**:用户的文字/聊天记录(本地)。
- **输出**:角色卡集合 + 关系图。挖的是「人生分叉点、没走的路、悔恨与设想、价值观、典型恐惧、口头禅、过去决策及其结局、你印象里的他人」。
- 原始文字 → 抽象画像的转换在本地完成,只有画像上云。

**分阶段流水线(借鉴 dot-skill,见记忆 colleague-skill-reference):** 不要单发,升级为 `analyzer(拆维度)→ builder(产出卡)→ validation → correction_handler(纠错回灌)`;名人卡另加 `research→synthesis→validation→audit` 治「虚构专业度」。

**persona 工艺纪律(必须遵守):**
- 卡的每条规则是**具体可执行的行为条件句**(「在 X 情况下做 Y」),**禁止形容词标签**。
- 证据不足的字段**显式标注「基于 {标签} 推断」**,不许编造。
- 卡带 **Correction 记录**(修订日志),纠错回灌时追加——这就是护城河。
- 当前 `scripts/distill_self_mirror.mjs` 是单发版,后续按此重写。

## 11. 技术栈(初步,待定)

- **桌面应用**:**Tauri**(Rust + Web 前端)——已定(2026-06-13)。体积小、更安全、更贴合本地加密存储与隐私定位。
- **模型后端**:**复用 Claude Code CLI(订阅鉴权)**——spawn `claude -p --output-format json/stream-json`,并把 `ANTHROPIC_API_KEY` 置空以走订阅、不按 token 计费(参考 `E:\ForStudy\Project\agentara` 的 `src/community/anthropic/claude-agent-runner.ts`)。已在本机验证(claude 2.1.175)。离线脚本用 Node `spawn`;Tauri 落地后改由 Rust `std::process::Command`。前提:本机装了并登录 Claude Code(个人/自用 OK;若将来分发/商用需重评鉴权与 ToS)。可选保留 API key 作备选后端。
- **本地存储**:加密的本地库(原始语料 + 角色卡 + 会话历史)。

## 12. MVP 切法

**已定(2026-06-13):从自动蒸馏管线切入**——用户认为这才是产品的魔法与护城河,优先攻坚。代价是门槛高、出效果慢,且需先解决**输入数据**问题(见 §13)。

- **第一步:钉死蒸馏管线的输入→输出契约**
  - 输出 = §2 的 Card schema + 关系图(已定)。
  - 输入 = 文字语料;v0 用**你自己的写作**(公众号/日记,最低风险),微信聊天记录留到第二刀。来源/格式见 §13。
- **管线分阶段**:原始语料(本地)→ 解析/分段 → 抽取(人生分叉点、价值观、典型恐惧、口头禅、过去决策及结局、你印象里的他人)→ 生成角色卡 + 关系图。原始文字→抽象画像在本地完成,只有画像上云。
- **推荐 v0 最小切片 = 「镜子里的我」**:从你自己的文字蒸出**一张** `self_mirror` 卡 → 进镜子模式做一次 1:1 引导对话。一箭三雕:① 跑通蒸馏管线(MVP 核心)② 只需一张卡、无需完整议会(最简)③ 自带验收——像不像你,一聊就知道。即「蒸馏出镜子自我」既是第一个要做的,也是检验蒸馏成没成的标尺。
- 后续:多卡辩论引擎 → 三模式收敛 → 关系切/名人卡/共享卡库 → 纠错养成闭环。
