# 把你的社交媒体扒成语料 —— 自助指南

> 给 MindStorms 用户：怎么把你自己在各平台写过的东西，变成蒸馏「镜子里的我」用的语料。
> **技术活儿不用你自己干**——本文给你「方法 + 可以直接粘贴给你自己的 AI（Claude Code / ChatGPT / Cursor…）的指令」，让 AI 替你写扒取脚本。
> 状态：指南（2026-06）。会随平台变化更新。

---

## 0. 一条铁律：只扒「你自己的」

- ✅ **你自己发的**：本人微博 / 小红书笔记 / 知乎回答 / 推文 / 朋友圈。这是你的 voice，是 MindStorms 真正要的，PIPL 也给你「查阅、复制自己数据」的权利。
- ❌ **别人的**：朋友的评论、对方那半边的私信、转发别人的原文。那是「我心里的他」隐私雷区（见 [CONCEPT.md](./CONCEPT.md) §9），**不扒、不出本机、不发表**。

记住：你不是在「爬别人」，你是在**把自己散落各处的话搬回家**。

---

## 1. 扒完存成什么（落地契约）

不管用哪种方法、哪个平台，**最终只要一种东西**：

> **你本人原话的纯文本，存成 `.md`，放进 `corpus/<平台名>/`。**

- 一条一段，或一篇一个文件都行；去掉 HTML 标签、话题链接、@、表情代码这些噪音。
- 例：`corpus/微博/weibo.md`、`corpus/小红书/notes.md`、`corpus/知乎/answers.md`、`corpus/twitter/tweets.md`。

然后接上现成管线（已有 `公众号 / 朋友圈 / 网易云` 在用同一套）：

1. 打开 `scripts/build_self_corpus.py`，找到「朋友圈 + 网易云」那段（`for label, pat in [...]`），**照葫芦画瓢加一行**你的新平台：

   ```python
   ("微博", r"...\corpus\微博\*.md"),
   ```

2. 跑蒸馏（见 `scripts/` 里的 distill 脚本 / [CONCEPT.md](./CONCEPT.md) §10、§12），新语料就进画像了。

> 把这条「落地契约」连同下面的指令一起发给你的 AI，它写出来的脚本就能直接对上管线。

## 2. 三种扒法（按「省事 / 稳」排序，先试上面的）

| | 方法 | 怎么回事 | 适合 | 抗封 |
|---|---|---|---|---|
| ① | **官方自助导出** | 平台自带「下载我的数据」，拿 ZIP/JSON，AI 只需写个解析器 | Twitter/X、微信公众号 | 最稳，零风控 |
| ② | **登录态 + 移动端接口** | 你浏览器登录 → 取 cookie → AI 写脚本带你的 cookie 翻页拉本人内容 | 微博、知乎 | 中（自己的号、慢拉基本安全） |
| ③ | **浏览器「边滚边抓」** | 用你登录好的页面，滚动加载，直接抓渲染出来的文字（油猴脚本 / 浏览器插件 / Playwright） | 小红书等强签名 App 系 | 高（读渲染结果，绕开签名） |

> 别让 AI 去逆向签名接口（小红书 `x-s`、知乎 `x-zse-96`）——天天变、维护地狱。③「抓 DOM」就是为了躲它。

## 3. 把活儿交给你的 AI：可直接粘贴的指令

### 通用模板（改中括号里的字就能用）

```
我要把我自己在【微博】上发过的内容导出，做成本地语料喂给一个叫 MindStorms 的项目。
请帮我写一个【Python】脚本：

【范围】只要「我本人原创」的正文——不要评论、不要转发别人的、不要别人的话。
【鉴权】我会先在浏览器登录，然后把我的 cookie 给你（或教我从浏览器导出）；用我自己的登录态抓。
【节奏】人肉节奏：每页之间 sleep 0.5~1.5 秒，串行、别并发，别触发风控；支持断点续传。
【清洗】去掉 HTML 标签、话题/链接、@某人、纯表情；保留我写的自然文字。
【输出】写到 corpus/微博/weibo.md，UTF-8，一条一段。这是给蒸馏管线吃的，越干净越好。

先别写代码：先告诉我①需要我提供什么（cookie 怎么取、放哪）②预计能抓到哪些字段，
我确认后你再给完整脚本。
```

### 针对方法的小改

- **① 官方导出（X）**：把模板的「鉴权/节奏」换成——
  `我已经在 设置→下载数据存档 拿到了 ZIP，里面有 tweets.js。帮我写个解析器，把我本人推文正文抽出来写到 corpus/twitter/tweets.md，跳过转推和纯链接。`
- **③ 边滚边抓（小红书）**：把模板的「Python 脚本」换成——
  `帮我写一个【浏览器油猴脚本 / Playwright 脚本】：在我已登录的本人主页，自动向下滚动加载我所有笔记，抓取每篇笔记的正文文字（读页面已渲染的 DOM，不要去调它的签名接口），导出成 corpus/小红书/notes.md。`

### 务必提醒你的 AI 的几个点（贴在指令后面）

```
注意：
- 这些平台有反爬，请用「我自己的登录态 + 慢速串行」，目标是低调拉我自己的数据，不是高并发爬。
- 不要尝试逆向它的加密签名接口；拿不到就退回到「在登录页面里读渲染好的内容」。
- 凡是抓到的别人的话/评论/对方私信，一律丢弃，不写进文件。
- 输出务必是纯文本 .md，放在 corpus/<平台>/ 下。
```

## 4. 各平台速查

| 平台 | 首选路子 | 关键点 |
|---|---|---|
| **Twitter / X** | ① 官方「下载数据存档」 | ZIP 里 `tweets.js` 全有，最省事，AI 只写解析器 |
| **微博** | ② 登录态 + `m.weibo.cn` 接口翻页 | 抓本人主页原创博文，慢拉；转发别人的丢掉 |
| **知乎** | ② 登录态 + 浏览器自动化 / 创作者后台导文章 | 匿名被签名挡死，登录态能过；拉本人回答+文章 |
| **小红书** | ③ 登录后「边滚边抓 DOM」 | 最硬，别碰签名接口；油猴/Playwright 抓渲染文字 |
| **微信公众号** | 已跑通 | 网页保存文章 HTML → `scripts/clean_wechat_html.py` |
| **朋友圈 / QQ空间** | 现成导出工具 → 转 `.md` | 朋友圈不在聊天库里；只留你自己发的那条，别带评论 |

## 5. 护栏（务必读）

1. **ToS**：自动访问多半违反平台条款，哪怕扒的是你自己。**自用 + 只落本地 + 不二次分发 + 慢速** → 实际风险低，但不是零。别拿去爬别人、别商用分发。
2. **风控 / 封号**：用你自己的登录态、人肉节奏（几百毫秒一页、串行）。一上来就猛拉容易触发风控甚至封号——**宁慢勿快，能续传**。
3. **本地优先**：原始扒下来的东西（尤其混进的他人信息）**死锁本机**。只有高度抽象的画像才上云（见 [CONCEPT.md](./CONCEPT.md) §9）。
4. **先小后大**：先让 AI 拉 10 条验证格式对不对、能不能进管线，再放开全量。
